## Проект

**Telegram-бот** на Python 3.12: ChatGPT + Midjourney + платежи + подписки.

**Стек:**
- `aiogram 2` — бот (long-polling, `main.py`)
- `FastAPI` — вебхуки оплат и MJ (`api_server.py`), отложенные уведомления (`notify_server.py`)
- `PostgreSQL` — база данных (хост `91.192.102.250`, БД `restored_db`)
- `APScheduler` — cron-задачи (ежедневное пополнение токенов и пр.)

**Структура:**
```
handlers/       — обработчики aiogram (users, admin, ref, sub, image)
keyboards/      — клавиатуры (user.py, admin.py)
utils/          — утилиты: db.py, ai.py, pay.py, mj_apis.py, history_token.py
states/         — FSM-состояния aiogram
middlewares/    — middleware (проверка подписки и пр.)
configs_files/  — конфиги nginx, systemd и т.д.
```

**Сервер:** `91.192.102.250` (`neuronbot.ru`), nginx + SSL.

## Правила для кода

- Всегда `python3` (не `python`)

- После изменения `.py` — проверять синтаксис: `python3 -m py_compile <файл>`

## Правило коммитов

После каждого изменения кода **обязательно** в конце ответа давать готовый текст коммита в код-блоке.

Формат (Conventional Commits, описание на русском):
```
<type>: Описание на русском
```

Типы: `fix` `feat` `chore` `refactor` `docs` `style` `test` `perf` `ci` `build` `revert`

Примеры:
```
feat: Добавлена функция форматирования секунд
fix: Исправлен баг в экспорте данных
refactor: Упрощена логика авторизации
```

**Триггеры:**
- Изменил / добавил код → дай текст коммита в код-блоке автоматически
- Пользователь написал «коммит» или «commit» → дай текст коммита в код-блоке
- Пользователь написал «автокоммит» → выполни `git add <файлы>`, `git commit -m "..."`, `git push` сам

Никогда не спрашивать «сделать коммит?» — просто делать по триггеру.

**Не коммитить:** `.env`, `tmp/`, `__pycache__/`, `.DS_Store`, `config.py`
